# 클라우드 파일 공유 시스템(미니드라이브) 프로젝트 관리 계획서 (Project Management Plan)

---

## 1. 개요 (Introduction)

### 1.1 문서의 목적

본 문서는 Mini Drive 개발에 대한 내용을 정리하여 프로젝트 진행에 있어 지표로 삼음을 목적으로 함

### 1.2 프로젝트 개요

- **정의**: 팀 내 모든 구성원이 언제 어디서든 파일을 업로드·다운로드·공유할 수 있는 웹 기반 클라우드 파일 관리 시스템
- **배경**: 기존 메신저·이메일을 통한 파일 공유 방식은 버전 혼재, 중복 파일 발생, 보안 취약 등의 문제가 빈번히 발생함. 이를 해결하기 위해 중앙화된 파일 저장소 구축을 기획함
- **주요 기능**
  - **파일 업로드/다운로드**: 다양한 형식의 파일을 웹 브라우저를 통해 손쉽게 업로드 및 다운로드
  - **폴더 구조 관리**: 계층형 폴더를 생성하여 파일을 체계적으로 분류 및 정리
  - **공유 링크 생성**: 외부 사용자에게도 특정 파일을 링크를 통해 공유 가능
  - **권한 관리**: 팀원별 읽기/쓰기/관리자 권한을 설정하여 접근을 제어
  - **파일 검색**: 파일명·확장자·업로드 날짜 등의 조건으로 빠르게 원하는 파일 탐색

### 1.3 프로젝트 목표

- 팀 내 파일을 하나의 중앙 저장소에서 일괄 관리
- 사용자가 파일 쉽게 업로드/다운로드
- 팀원 간 파일 공유 및 협업 환경 구축
- 공유 링크를 통한 외부 사용자 접근 지원

### 1.4 용어 정의

- **미니 드라이브**: 본 프로젝트에서 구축하는 클라우드 파일 공유 시스템의 명칭
- **공유 링크**: 외부 사용자가 파일을 다운로드할 수 있도록 생성된 고유 URL

### 1.5 참조 문서

- 소프트웨어 공학 강의 자료 : Ch.05 프로젝트 관리
- Requirement Specification

## 2. 개발 계획 (Development Plan)

### 2.1 프로세스 모델

- **SCRUM 모델**: 팀원 간 의사소통 원활하게 하여 사용자의 불명확하고 자주 변경되는 요구사항을 빠르게 반영 가능

### 2.2 작업 분할도 (WBS)

| 작업 명칭 | 상세 활동 (Task) |
|---|---|
| 요구분석 | 유사 서비스 벤치마킹, 사용자 요구사항 도출, 기능 목록 정의 및 우선순위 설정 |
| 시스템 설계 | ERD 및 DB 스키마 설계, REST API 명세 작성, 전체 시스템 아키텍처 설계 |
| 시스템 구현(코딩) | 프론트엔드 UI 구현, 백엔드 API 서버 구축, 파일 저장소 연동 및 기능 통합 |
| 테스트 및 검증 | 단위 테스트, 통합 테스트, 사용자 시나리오 기반 인수 테스트 수행 |
| 배포 및 인도 | 서버 배포, 사용자 매뉴얼 작성, 최종 결과 보고서 제출 |

### 2.3 개발 내용

- **플랫폼**: 웹 서비스 기반 (React + Spring Boot 등)
- **데이터베이스**: MySQL (파일 메타데이터 및 사용자 정보 저장)

| 팀장(PM) | * | 프로젝트 일정 관리, 문서 총괄, 팀원 간 조율 및 최종 검토 |
