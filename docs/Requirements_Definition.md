## 1. 서론

### 1.1 문서 목적 및 범위

본 문서는 클라우드 파일 공유 시스템(미니드라이브) 개발 프로젝트에서 요구되는 사항이 무엇인지 조사하고 정의하는 문서이다.  
기능적 요구사항, 비기능적 요구사항, 인터페이스 요구사항을 포함하며, 개발팀 전원이 동일한 기준으로 시스템을 이해하고 구현할 수 있도록 한다.

### 1.2 프로젝트 개요

#### 1.2.1 프로젝트 정의

누구나 웹 브라우저만으로 파일을 업로드·다운로드·공유할 수 있는 클라우드 기반 파일 관리 시스템을 구축한다.  
팀원 간 협업을 위한 폴더·권한 관리와 외부 공유 링크 기능을 제공하며, React + Spring Boot + MySQL 기술 스택을 기반으로 개발한다.

#### 1.2.2 주요 기능 설명

| ID | 설명 |
|---|---|
| F01 | 사용자는 웹 브라우저를 통해 파일을 업로드 및 다운로드할 수 있다 |
| F02 | 사용자는 계층형 폴더를 생성하여 파일을 체계적으로 분류할 수 있다 |
| F03 | 관리자는 팀원별 읽기/쓰기/관리자 권한을 설정할 수 있다 |
| F04 | 사용자는 파일명·확장자·업로드 날짜로 파일을 검색할 수 있다 |
| F05 | 사용자는 외부 공유용 링크를 생성하여 비로그인 사용자에게 파일을 공유할 수 있다 |

### 1.3 용어 정의

| 용어 | 설명 |
|---|---|
| 공유 링크 | 로그인 없이 특정 파일에 접근할 수 있도록 발급되는 만료 기한이 설정된 고유 URL |
| 권한 등급 | 파일 및 폴더에 대한 접근 수준 (읽기 전용 / 편집 가능 / 관리자) |
| 메타데이터 | 파일명, 확장자, 크기, 업로드 날짜 등 파일의 속성 정보 |
| REST API | 클라이언트와 서버 간 데이터를 주고받기 위한 HTTP 기반 인터페이스 규약 |

## 2. 요구사항

### 2.1 기능적 요구사항

#### 2.1.1 회원 및 인증 기능

| 분류 | 요구사항 | 우선순위 |
|---|---|---|
| FR-001 | 사용자는 이메일, 비밀번호를 입력하여 회원가입을 할 수 있다 | 상 |
| FR-002 | 사용자는 이메일, 비밀번호를 입력하여 로그인한다 | 상 |
| FR-003 | 사용자는 로그아웃할 수 있다 | 상 |

#### 2.1.2 파일 관리 기능

| 분류 | 요구사항 | 우선순위 |
|---|---|---|
| FR-004 | 사용자는 단일 또는 다중 파일을 업로드할 수 있다 | 상 |
| FR-005 | 사용자는 업로드된 파일을 다운로드할 수 있다 | 상 |
| FR-006 | 사용자는 파일을 삭제할 수 있다 | 상 |
| FR-007 | 사용자는 파일명을 변경할 수 있다 | 중 |
| FR-008 | 시스템은 업로드된 파일의 메타데이터(크기, 확장자, 날짜)를 DB에 저장한다 | 상 |

#### 2.1.3 폴더 관리 기능

| 분류 | 요구사항 | 우선순위 |
|---|---|---|
| FR-009 | 사용자는 폴더를 생성, 수정, 삭제할 수 있다 | 상 |
| FR-010 | 사용자는 파일을 원하는 폴더로 이동할 수 있다 | 중 |

#### 2.1.4 공유 링크 기능

| 분류 | 요구사항 | 우선순위 |
|---|---|---|
| FR-011 | 사용자는 특정 파일에 대한 공유 링크를 생성할 수 있다 | 상 |
| FR-012 | 공유 링크에는 만료 날짜를 설정할 수 있다 | 중 |
| FR-013 | 비로그인 사용자는 공유 링크를 통해 파일을 다운로드할 수 있다 | 상 |

#### 2.1.5 권한 관리 기능

| 분류 | 요구사항 | 우선순위 |
|---|---|---|
| FR-014 | 관리자는 팀원에게 읽기 전용 / 편집 가능 / 관리자 권한을 부여할 수 있다 | 상 |
| FR-015 | 권한이 없는 사용자는 해당 파일 또는 폴더에 접근할 수 없다 | 상 |

#### 2.1.6 파일 검색 기능

| 분류 | 요구사항 | 우선순위 |
|---|---|---|
| FR-016 | 사용자는 파일명을 키워드로 검색할 수 있다 | 상 |
| FR-017 | 사용자는 확장자 또는 업로드 날짜 범위로 파일을 필터링할 수 있다 | 중 |

### 2.2 비기능적 요구사항

#### 1) 운영 환경에 대한 요구사항

| 분류 | 요구사항 | 우선순위 |
|---|---|---|
| NFR-001 | 시스템은 Chrome, Edge, Safari 등 주요 웹 브라우저에서 동작한다 | 상 |
| NFR-002 | 시스템은 모바일 화면 크기에서도 정상적으로 레이아웃이 표시된다 (반응형) | 중 |

#### 2) 성능 요구사항

| 분류 | 요구사항 | 우선순위 |
|---|---|---|
| NFR-003 | 시스템은 모든 사용자 요청에 대해 3초 내로 응답한다 | 상 |
| NFR-004 | 단일 파일 업로드 최대 용량은 500MB로 제한한다 | 상 |
| NFR-005 | 파일 목록 및 검색 결과는 1초 내로 반환된다 | 중 |

#### 3) 보안 요구사항

| 분류 | 요구사항 | 우선순위 |
|---|---|---|
| NFR-006 | 시스템은 모든 사용자의 비밀번호를 암호화(해시)하여 저장한다 | 상 |
| NFR-007 | 공유 링크는 만료 기한이 지나면 자동으로 접근이 차단된다 | 상 |
| NFR-008 | 권한 없는 사용자의 파일 접근 시도는 로그로 기록된다 | 중 |

#### 4) 문화 및 정책적 요구사항

| 분류 | 요구사항 | 우선순위 |
|---|---|---|
| NFR-009 | 시스템은 한국어 UI를 기본으로 제공한다 | 상 |

### 2.3 인터페이스 요구사항

|---|---|---|
