# 대상 시스템 품질 요소 추정하기 (프로젝트 II)

##  사용자가 원하는 기대치는 무엇인가?  
- 별도의 학습 없이도 직관적으로 화면을 조작할 수 있는 것을 원한다.  
- 파일 업로드/다운로드, 공유, 권한 설정 기능이 정확하게 동작하기를 원한다.  
- 파일이 유실되지 않고 안정적으로 저장되기를 원한다.  
- 지연 시간 없이 결과가 즉시 나타나기를 원한다  
- 내 개인정보나 결제 정보가 외부로 유출되지 않도록 안전하게 보호된는 것을 원함  
- 오류 상황에서도 시스템이 중단되지 않고 정상적으로 동작하기를 원함

##  개발자 및 유지보수 담당자가 원하는 기대치는 무엇인가?
- 개발자와 유지보수 담담자는 나중에 코드를 수정하거나 기능을 추가하기가 얼마나 편한가를 중요하게 본다  
- 새로운 기능을 추가할 때 기존 시스템의 구조를 크게 바꾸지 않는 유연한 구조를 원한다  
- 코드가 읽기 쉽고 이해하기 쉬운 구조로 작성되기를 원한다  
- 요구사항과 코드 간의 관계를 추적할 수 있기를 원한다  
- 운영체제가 바뀌거나 서버 환경이 변하더라도 큰 수정 없는 시스템을 원한다.

##  개발 조직의 관리자가 원하는 기대치는 무엇인가?
- 최소한의 인력과 시간으로 목표로 한 기능을 구현하기를 원한다  
- 시장의 요구사항에 맞춰 정해진 데드라인을 엄격히 준수하기를 원한다  
- 모듈이나 컴포넌트를 다른 프로젝트에서도 사용해 장기적인 비용을 절감하기를 원한다.  
- 시스템의 상태와 품질을 쉽게 확인할 수 있기를 원한다.
